用hexo在github上构建博客——Mac版

第0篇博客

历尽千辛万苦终于搭建好了这么一个简陋的博客，其实原本应该半小时内就可以实现的工作，结果耗了我4个小时，主要有两方面原因，其中之一是，我真的只是个小白；而第二呢，也是最最主要的原因，就是网上的教程良莠不齐，难得找到一两篇良的，结果都是基于windows系统的，连Linux都有，于是我当初就下定决心，待我搭建好了博客，得将我这苦逼心路历程都写下来，希望能帮助有同样建博客需求的Mac用户少走一些弯路。事不宜迟，即将开始：

安装xcode

这个比较方便，直接在Mac App Store上搜索下载就ok，下载好了以后，启动并进入 Preferences -> Download -> Command Line Tools -> Install 安装命令行工具。

安装Git

使用Homebrew来安装即可。启动终端机，将下列代码：
ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"黏贴进去，回车，稍等片刻，完成。

安装Node.js

官网推荐的安装方式是使用nvm。以下安装脚本二选一即可：
cURL:
$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh
Wget:
$ wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh
安装完成后，重启终端机并执行下列命令即可：
$ nvm install
关于nvm其他安装方式的详情可点击：https://github.com/creationix/nvm

安装Hexo

所有准备工作已搞定，终于来到高潮啦！正式可以使用npm来安装Hexo！只需输入：
$ npm install -g hexo-cli
若上述代码不能使用，可试试输入下面的：
$ npm install -g hexo
剩下的只是等待，让它自动安装完毕即可。

建站

安装Hexo后，又要再次打开终端机啦，我们需要让Hexo在指定文件夹中新建所需要的文件哟，建议是直接在finder中的个人专属新建一个名为“hexo”的文件夹即可，然后输入下列代码：
$ cd /hexo这里的hexo是表示新建的文件夹名字 ``$ hexo init ``$ npm install
新建完成以后，指定文件夹的目录如下：

├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
| ├── _drafts
| └── _posts
└── themes

其中网站具体的配置，是打开config.yml 直接修改的。

具体的配置方法，请前往http://hexo.io/zh-cn/docs/configuration.html慢慢参详，都是比较简单的啦。

部署github

下面就到了如何将hexo构建的静态博客部署到github上啦，现在已经完成了50%啦，keep moving！！下面是具体做法：

  1. 注册github；
  2. 新建仓库repository，且仓库名必须为yourUserName.github.com 或者yourUserName.github.io；
  3. 最最难最最麻烦的一步，生成并添加SSH key。下面详细来说说：

Step 1: Check for SSH keys

首先呢，不要太冲动急着生成SSH keys，首先得查一查自己的Mac里有没有现成的嘛。如何查询呢，启动终端机，输入以下代码并回车：
$ ls -al /.ssh

Step 2: Generate a new SSH key

如果木有的话呢，就要亲自来生成啦！输入以下代码，用自己的github注册邮箱来生成：
ssh-keygen -t rsa -C "your email@example.com"
然后会出现以下代码：
Enter file in which to save the key (/Users/you/.ssh/id rsa):官方建议是直接按enter即可；

接着你要输入一个密码：
Enter passphrase (empty for no passphrase): ``Enter same passphrase again:ps: 密码别起得太简单哟！

输入结束以后，你会获得一个指纹啦，id啦，还有你的SSH key啦，具体长啥样呢？可以参考下图：
Your identification has been saved in /Users/you/.ssh/id rsa. ``Your public key has been saved in /Users/you/.ssh/id rsa.pub. ``The key fingerprint is: ``01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your email@example.com

Step 3: Add your key to the ssh-agent

现在需要把SSH key 添加到ssh-agent啦，首先需要确认一下ssh-agent是否已启用：
$ eval "$(ssh-agent -s)" ``Agent pid 59566
把生成的SSH key添加至ssh-agent：
$ ssh-add /.ssh/id rsa

Step 4: Add your SSH key to your account

登录你的github账户，前往Account settings -> SSH Keys -> Add SSH Key， 把刚生成的文件：idrsa.pub里面的内容一个字符不漏地复制到key里面，然后点击 Add key，最后输入你的github的密码即可。

Step 5: Test the connection

最后来测试一下是否成功连接啦，打开终端机，输入：
$ ssh -T git@github.com
然后会看到一大串警告，忽略，直接输入： yes ，回车；

如果你看到以下信息：
Hi username! You've successfully authenticated, but GitHub does not provide shell access.
且里面的username是你的账户名字，那证明链接大成功！！

生成静态页面

启动终端机，输入：
$ cd /hexo这里的hexo是表示安装hexo的文件夹名字 ``$ hexo generate

本地启动

执行以下命令，启动本地服务，即可预览文章，进行调试啦
$ hexo server
浏览器键入 http://localhost:4000 可观察到效果。

将hexo构建的博客与github建立关联

  1. 安装 hexo-deployer-git 。终端机输入：
    $ npm install hexo-deployer-git --save
  2. 修改config.yml设置：
    用xcode打开文件，按以下项目修改：
    deploy: `` type: git `` repo: <repository url> 从github账户可查询到，建议使用https `` branch: 一般来说，是master，也可从github账户查询到

  3. 保存退出；

  4. 在终端机输入以下代码：
    $ hexo deploy如果显示done的话，那就圆满啦！！我看到成功信息的时候，真有种想哭的冲动啊！

结尾

到了这里，基本上就只剩下挑个好的markdown app来尽情地写文章啦，顺便可以按照hexo官网来尽情个性化属于你的博客，废话不多说，剩下的东西留给各位客官们自己好好探寻啦～再会～
